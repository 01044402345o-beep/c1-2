### 씬 1 (0~2초) — 문제 제시
- 씬 번호/길이: Scene 01 / 2.0초
- 목표 메시지: 아직도 손으로 기록을 쓰고 있는 현실을 공감시킨다
- 화면 구성(구도/피사체/배경/텍스트): 손 클로즈업(탑뷰/45도) / 60대 여성의 손·기록지·펜 / 미니멀 사무 책상, 베이지·화이트 / 하단 자막 있음
- 내레이션(카피): "매일 반복되는 다량의 기록..."
- 사용 도구(목적): 이미지=GPT Image2, 비디오=OpenAI Sora 2(I2V), 오디오=ElevenLabs(나레이션)
- 입력 프롬프트(원문): `책상에 앉아 종이 급여기록지를 손으로 작성하는 60대 한국인 여성. 따뜻하고 미니멀한 사무실 내부, 부드러운 자연 채광, 약간 피곤한 표정, 손과 기록지를 클로즈업, 베이지와 화이트 톤, 시네마틱, 35mm 필름 질감`
- Veo 변환 프롬프트: `카메라가 종이 기록지를 작성하는 여성의 손을 천천히 확대한다. 펜이 기록지를 자연스럽게 작성한다. 따뜻한 자연광이 책상 위를 부드럽게 비춘다. 따뜻한 미니멀 사무실, 조용하고 약간 무거운 분위기, 베이지와 화이트 톤, 시네마틱, 부드러운 카메라 움직임, 35mm 필름 느낌, 배경은 변경되지 않는다.`
- 출력 결과 요약: 손과 기록지 클로즈업, 따뜻한 자연광의 미니멀 사무 공간 확보(씬1.png)
- 결과: 최종 영상 0~2초 구간

### 씬 2 (2~4.5초) — 감정 고조
- 씬 번호/길이: Scene 02 / 2.5초
- 목표 메시지: 반복되는 서류 작업의 피로감에 감정적으로 이입시킨다
- 화면 구성: 상반신 미디엄샷 / 지친 표정의 60대 여성, 쌓인 서류 / 흐릿한 미니멀 사무실, 따뜻한 오후 빛 / 하단 자막 있음
- 내레이션(카피): "퇴근 후에도 끝나지 않는 서류 작업"
- 사용 도구(목적): 이미지=GPT Image2, 비디오=OpenAI Sora 2(I2V), 오디오=ElevenLabs(나레이션)
- 입력 프롬프트(원문): `서류 더미에 둘러싸인 책상에 앉아 눈을 비비거나 손에 얼굴을 기댄 피곤한 60대 한국인 여성. 늦은 오후의 따뜻한 햇살, 미니멀한 사무실, 부드럽게 흐려진 배경, 감정이 담긴 클로즈업, 따뜻한 베이지 톤, 시네마틱 사진.`
- Veo 변환 프롬프트: `피곤한 60대 한국인 여성을 정적인 클로즈업으로 촬영한다. 천천히 숨을 내쉬고 눈을 부드럽게 비빈다. 앞쪽의 서류가 살짝 흔들린다. 늦은 오후의 따뜻한 햇살이 은은하게 비춘다. 감정적이고 피로한 분위기, 미니멀한 사무실 배경, 따뜻한 베이지 톤, 시네마틱, 35mm 필름 느낌, 배경은 변경되지 않는다.`
- 출력 결과 요약: 지친 표정과 쌓인 서류가 함께 담긴 감정적 장면 확보(씬2.png)
- 결과: 최종 영상 2~4.5초 구간

### 씬 3 (4.5~7초) — 해결책 등장
- 씬 번호/길이: Scene 03 / 2.5초
- 목표 메시지: 말 한마디로 기록이 완성되는 순간을 직관적으로 보여준다
- 화면 구성: 얼굴+스마트폰 미디엄 클로즈업 / 스마트폰에 말하는 60대 여성, 화면에 완성된 기록지 / 밝고 깔끔한 미니멀 사무실 / 하단 자막 있음
- 내레이션(카피): "말하는 순간, 기록이 완성됩니다"
- 사용 도구(목적): 이미지=GPT Image2, 비디오=OpenAI Sora 2(I2V), 오디오=ElevenLabs(나레이션)
- 입력 프롬프트(원문, 수정 후): `스마트폰에 자연스럽게 이야기하는 60대 한국 여성, 편안하고 미소 띤 표정, 밝고 깔끔한 사무실 배경, 디지털 돌봄 기록 양식이 작성된 스마트폰 화면, 부드럽고 따뜻한 조명, 깨끗한 화이트 및 베이지 톤의 인테리어, 얼굴과 스마트폰을 강조한 클로즈업 샷, 희망적인 분위기, 영화 같은 느낌`
- Veo 변환 프롬프트: `카메라가 스마트폰에 말하는 60대 한국인 여성을 향해 천천히 다가간다. 스마트폰 화면에는 완성된 기록지가 나타난다. 표정이 안도감과 미소로 바뀐다. 따뜻한 자연광, 밝은 미니멀 사무실, 희망적이고 따뜻한 분위기, 베이지와 화이트 톤, 시네마틱, 35mm 필름 느낌, 배경은 변경되지 않는다.`
- 출력 결과 요약: 스마트폰에 말하며 기록이 완성되는 밝고 희망적인 장면 확보(씬3.png)
- 결과: 최종 영상 4.5~7초 구간

### 씬 4 (7~8초) — 브랜드 각인
- 씬 번호/길이: Scene 04 / 1.0초
- 목표 메시지: In Voice 브랜드를 신뢰감 있게 각인시키고 행동을 유도한다
- 화면 구성: 제품 미니멀 센터샷 / 완성된 기록지 화면이 보이는 스마트폰 / 순백·베이지 미니멀 / 브랜드명 "In Voice" + 슬로건 "말하면, 기록됩니다" + CTA "지금 시작하세요"
- 내레이션(카피): "In Voice — 말하면, 기록됩니다"
- 사용 도구(목적): 이미지=GPT Image2(브랜드 제품샷), 비디오=OpenAI Sora 2(I2V), 오디오=ElevenLabs(나레이션)
- 입력 프롬프트(원문): `순백의 미니멀 배경, 완성된 급여기록지가 표시된 스마트폰, 측면의 부드러운 따뜻한 조명, 심플하고 고급스러운 제품 연출, 사람은 등장하지 않음, 스마트폰 화면에 초점, 베이지와 화이트 색상, 전문적이고 신뢰감 있는 분위기, 프리미엄 광고 사진.`
- Veo 변환 프롬프트: `완성된 급여기록지가 표시된 스마트폰이 미니멀한 흰색 책상 위에 놓여 있다. 카메라는 천천히 뒤로 이동하며 깔끔한 작업 공간을 보여준다. 측면의 따뜻한 조명이 은은하게 비춘다. 화면은 끝까지 선명하게 보인다. 전문적이고 신뢰감 있는 차분한 분위기, 순백과 베이지 톤, 미니멀 제품 광고 스타일, 시네마틱, 배경은 변경되지 않는다.`
- 출력 결과 요약: 완성된 기록 화면이 보이는 스마트폰 미니멀 제품샷 확보(씬4.png)
- 결과: 최종 영상 7~8초 구간(브랜드 장치 구간)

### 프롬프트 수정 전/후 (씬 3 기준)

**수정 전**: `스마트폰에 대고 말하며 미소 짓는 한국 여성, 사무실 배경, 영화 같은 분위기`

**문제점**: 연령대 미지정으로 20~30대 여성이 생성됨(타겟 불일치) / 스마트폰 화면 내용 미지정(빈 화면 생성) / 배경 톤 미지정(차갑고 어두운 사무실) / 감정 묘사 부족(표정이 광고 톤과 불일치)

**수정 후**: `스마트폰에 자연스럽게 이야기하는 60대 한국 여성, 편안하고 미소 띤 표정, 밝고 깔끔한 사무실 배경, 디지털 돌봄 기록 양식이 작성된 스마트폰 화면, 부드럽고 따뜻한 조명, 깨끗한 화이트 및 베이지 톤의 인테리어, 얼굴과 스마트폰을 강조한 클로즈업 샷, 희망적인 분위기, 영화 같은 느낌`

| 수정 항목 | 수정 이유 | 결과 변화 |
|----------|----------|----------|
| `60대 여성` 추가 | 타겟(60대 시설장) 일치 | 연령대 맞는 인물 생성 |
| `디지털 케어 기록 양식이 표시된 스마트폰 화면` 추가 | 핵심 기능 시각화 필요 | 기록지 화면이 보이는 폰 생성 |
| `미니멀하고 밝은 사무실`, `화이트&베이지` 추가 | 톤앤매너 통일 | 따뜻하고 밝은 배경으로 개선 |
| `희망적인 분위기` 추가 | 전환 씬의 감정 톤 명시 | 희망적·긍정적 표정으로 개선 |

## 5. 서사 / 메시지 (광고 완성도)

- 구조: **기승전결** — 문제(손글씨 기록의 반복, 씬1) → 감정 고조(퇴근 후에도 끝나지 않는 서류 작업, 씬2) → 전환/해결(음성으로 기록 완성, 씬3) → 브랜드 각인(씬4).
- 마지막 1초(씬4): 브랜드명 "In Voice" + 슬로건 "말하면, 기록됩니다" + CTA "지금 시작하세요" = 브랜드 인지 장치("마지막 3~5초 브랜드 장치" 기준 대비 8초 광고 특성상 마지막 1초에 압축 배치, 씬3 후반부터 톤을 밝게 전환해 각인 여운을 확보).
- 톤 아크: 씬1~2(피곤하고 무거운 베이지톤) → 씬3(밝고 희망적인 화이트·베이지톤) → 씬4(순백·베이지 브랜드톤)로 점층 전환하여 감정 곡선과 브랜드 각인을 연결.

## 6. 동작 구조 및 설계 (제작 파이프라인)

```
[기획 의도] 브랜드/타겟/메시지 확정(60대 여성 시설장 · 음성인식 자동기록)
  → [프롬프트 설계] 씬별 프롬프트(스타일 레퍼런스 고정: 베이지·화이트, 부드러운 자연광)
  → [이미지 생성] GPT Image2 → 키프레임 4종 확정(씬1~4.png, 스토리보드 잠금)
  → [비디오 변환] OpenAI Sora 2(I2V) → 정지이미지 기반 카메라 무빙(배경 불변 조건 명시)
  → [오디오] ElevenLabs 내레이션(4카피)
  → [통합 편집] ffmpeg: concat(2+2.5+2.5+1) · 오디오 mux · 9:16 변환
  → [출력] c1-2.mp4 (+ 보너스 9:16)
```

- 설계 의사결정: 이미지 단계에서 인물 연령대·표정·배경 톤을 프롬프트로 먼저 고정한 뒤, 씬3에서 발견된 문제(연령 불일치·빈 화면 등)를 프롬프트 수정으로 해결하고 이를 전 씬에 소급 적용해 일관성을 확보. Veo 변환 프롬프트에 "배경은 변경되지 않는다"를 공통 지시어로 명시해 I2V 변환 시 원본 이미지의 인물·배경 정체성을 유지.

## 7. 핵심 기술 원리 (도구별 역할/강점·약점)

- **텍스트→이미지(GPT Image2)**: 디퓨전 기반. 강점=인물 표정·연령대·조명 지시어 반영 정밀도 높음, 한국인 인물 표현 자연스러움. 약점=제품 화면 내 텍스트(기록지 UI 등) 초소형 디테일은 흔들릴 수 있어 핵심 요소(브랜드명·슬로건)만 배치.
- **텍스트/이미지→비디오(OpenAI Sora 2, I2V)**: 확산-트랜스포머 기반. 강점=정지이미지를 입력으로 배경·인물 정체성을 유지한 채 미세한 카메라 무빙·표정 변화를 자연스럽게 생성. 약점=긴 동작(다단계 제스처)에는 불안정 → 씬당 1~2개의 단순 동작(펜 작성, 숨 내쉬기, 미소 전환 등)으로 프롬프트 제한.
- **텍스트→음성(ElevenLabs)**: 강점=빠른 무저작권 한국어 보이스 생성, 8초 분량 4카피에 적합. 약점=미세한 감정 톤 조정 한계 → 카피를 짧고 명확하게 구성.
- **일관성 유지**: 스타일 레퍼런스(베이지·화이트, 자연광) 고정 + Veo 변환 프롬프트 공통 지시어("배경은 변경되지 않는다") + 톤 아크(무거움→밝음→브랜드톤) 설계로 씬 간 화풍·인물 불일치 방지.

## 8. 심층 인터뷰 대응

- **Q. 이미지/비디오/오디오 도구의 역할과 차이?** → §7. 이미지=정지 키비주얼(GPT Image2), 비디오=카메라 무빙·표정 변화(Sora 2 I2V), 오디오=내레이션 레이어(TTS). 각 단계 출력이 다음 입력이 됨(스틸→I2V, 영상+음성→mux).
- **Q. 같은 요구를 여러 도구로 시도한 결과 차이?** → 씬3 이미지 프롬프트를 수정 전/후로 각각 생성 비교(§2 프롬프트 수정 전/후). 연령대·화면 내용·톤 지시어 유무에 따라 결과가 크게 달라짐을 확인, 수정 후 버전을 채택. 비디오 대체 도구(Kling·Pika·Runway) 대비 Sora 2가 배경 불변 조건 준수도가 높아 최종 채택.
- **Q. 미디어 소스를 해상도/비율/톤 관점으로 어떻게 통합?** → 전 씬 1080p·16:9로 통일(ffmpeg scale/crop), 스타일 레퍼런스(베이지·화이트)로 톤 고정, 무거움→밝음→브랜드톤의 톤 아크로 색 연결. 세로형은 센터크롭 9:16 별도 출력.
- **Q. 크레딧/대기열 제약 대응?** → 이미지로 스토리보드를 먼저 확정한 뒤 씬별 I2V 변환은 짧은 단위(1~2.5초)로 분할 생성해 재시도 비용을 최소화. 8초 총 분량을 4씬으로 나눠 씬별 재생성 시에도 전체를 다시 만들지 않도록 설계.
- **Q. 60초 버전을 15초로 줄여야 한다면 씬을 어떻게 재구성?** → ① 핵심 메시지 1개(음성→자동기록)만 남기고 감정 고조 컷(씬2)을 축약. ② 씬당 3초 내 단축, 전환 컷 위주로 재편. ③ 내레이션은 핵심 카피 1줄 + 엔드카드 CTA만 유지. ④ 기승전결 4씬을 문제(1)/해결(1)/브랜드(1) 3씬으로 머지, 브랜드 장치(로고·슬로건·CTA)는 마지막 2~3초 필수 보존. ⑤ 자막 키워드 1개씩 노출로 정보 밀도 보강.

## 제약/환경 기록

- 도구·채널·설정: 이미지=GPT Image2(웹) / 비디오=OpenAI Sora 2(I2V, 정지이미지 입력) / 음성=ElevenLabs / 편집=ffmpeg.
- 소스: 전부 생성형 AI 결과물(직접 촬영·유료 스톡 미사용). 딥페이크·유해·개인정보 콘텐츠 없음. 등장 인물은 실존 인물이 아닌 AI 생성 가상 인물.
- 편집 SW(ffmpeg)는 컷 연결·오디오 mux·비율 변환 등 통합 편집 용도로만 사용(생성은 전부 AI 도구).
